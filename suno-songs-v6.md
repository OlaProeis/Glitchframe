# Suno Songs v6.1 — Bass Music, Light Vocals, Force-Controlled Arrangement

> Five tracks built with **skill v6.1** — the arrangement force-control toolkit.
> Each uses the **two-layer Structure Prompting** technique:
> - **Style field** carries the sonic world + an explicit `arrangement: ...` line naming sections, bar counts, drop count, and drop 2 relationship to drop 1 + a closing mirror anchor.
> - **Lyrics field** carries bracket tags with **bar-count suffixes**, `[Drop]` written **twice** for two drops, identical parenthetical text on both drops + "one added layer", `[Silence: 1s]` pre-drop impact cut, `[Break]` + `[Build]` mandatory between drops.
>
> Vocals are present but restrained: sparse ad-libs on verses, choir-voiced harmonics on chord tones during drops, no full verse lines.

Paste Style into Suno's **Style of Music** box, Lyrics into **Lyrics** box. Model: v5.5, Custom Mode, Magic Wand OFF.

## Generation Checklist (v6.1)

Before every Create click:

**Style field**
- [ ] Arrangement line present as final descriptor (`arrangement: ...`)
- [ ] Drop count stated explicitly (`two drops only`, `no bridge`)
- [ ] Drop 2 relationship stated (`identical plus one [layer] added`)
- [ ] Mirror anchor on the end (`— [core genre]`)
- [ ] No brackets in Style (they glitch — use plain words)
- [ ] Length 300–450 chars

**Lyrics field**
- [ ] `[Drop]` appears **exactly twice** (one per intended drop)
- [ ] Both `[Drop]` parentheticals have **identical base text** + "plus one [layer] added" on the second
- [ ] `[Break]` + `[Build]` always between the two drops — never adjacent
- [ ] `[Silence: 1s]` before first drop only (not second)
- [ ] Bar-count suffixes on `[Intro]`, `[Drop]`, `[Break]`, `[Build]`, `[Outro]`
- [ ] `[Outro 4]` has decay parenthetical
- [ ] Ends with `[End][End][End]`

**Suno UI**
- [ ] Model: **v5.5**
- [ ] Custom Mode: **ON**
- [ ] Magic Wand: **OFF** (silently rewrites Style in 5.5)
- [ ] Weirdness: **mid-high**
- [ ] Style Influence: **mid-high**

## Recovery Workflow — If Drop 2 Drifts

Even with the force-control toolkit, Suno 5.5 will occasionally ignore the second drop match. When it does, **don't regenerate 10 times**. Switch to the Song Editor:

1. Listen to the generation. Identify the best drop (usually drop 1).
2. Open in Song Editor.
3. Locate the bar position at the END of the winning drop.
4. Use **Extend** from that point.
5. Style field: **identical** to the original (don't edit).
6. Lyrics field: trimmed to continuation only:
   ```
   [Break 4]
   (strip to sub and snare tail)

   [Build 4]
   (riser builds, kick drops out)

   [Drop 16]
   (same bass pattern plus one [layer] added)

   [Outro 4]
   (element-by-element decay, bass drops out first)

   [End][End][End]
   ```
7. Generate. The Extend **inherits** the first drop's timbre, groove, and vocal identity because it's literally a continuation of that audio. This is the most reliable way to get a matched second drop.

This workflow costs 3 credits and takes 2 minutes. Regenerating in hope of a lucky match costs 20+ credits and usually fails. Use the Extend.
